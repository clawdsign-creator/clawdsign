# Contributing to ClawdSign

## Ways to Contribute
- Report bugs
- Suggest features
- Improve documentation
- Submit code

## Development Setup
1. Fork repository
2. Clone: `git clone ...`
3. Install: `npm install`
4. Run: `npm start`

## Code Style
- Use 2 spaces for indentation
- Follow ESLint rules
- Add comments for complex logic

## Commit Messages
Format: `type: description`

Types:
- feat: New feature
- fix: Bug fix
- docs: Documentation
- style: Formatting
- refactor: Code restructuring

## Pull Request Process
1. Create branch: `feature/amazing-feature`
2. Make changes
3. Test thoroughly
4. Create PR with clear description
5. Wait for review

## Code of Conduct
Be respectful, inclusive, and professional.
